# WBUK AI Triage Platform - Deployment Guide

## Quick Install (Ubuntu 24.04)

Run this single command on a fresh Ubuntu 24.04 server:

```bash
curl -sSL https://raw.githubusercontent.com/<YOUR_USERNAME>/wbuk-triage/main/deploy/install.sh | sudo bash
```

## What Gets Installed

The install script sets up:

- **Docker & Docker Compose** - Container orchestration
- **MongoDB 7** - Database for cases, sessions, users
- **MinIO** - S3-compatible object storage for evidence files
- **Backend** - FastAPI server with AI triage engine
- **Frontend** - React web application with WBUK branding
- **Nginx** - Reverse proxy and static file serving

## Requirements

- Ubuntu 24.04 LTS (other versions may work)
- Minimum 2GB RAM
- 20GB disk space
- Root/sudo access
- Open ports: 80 (HTTP), 443 (HTTPS), 9001 (MinIO console)

## Configuration

After installation, edit `/opt/wbuk-triage/.env` to configure:

```bash
# Public URL
PUBLIC_URL=https://your-domain.com

# MongoDB credentials
MONGO_USER=wbukadmin
MONGO_PASSWORD=<auto-generated>

# MinIO credentials
MINIO_USER=minioadmin
MINIO_PASSWORD=<auto-generated>

# JWT Secret (for authentication)
JWT_SECRET=<auto-generated>

# Emergent LLM Key (required for AI features)
EMERGENT_LLM_KEY=<your-key>
```

### Getting an Emergent LLM Key

1. Visit https://emergent.sh
2. Create an account or sign in
3. Go to Profile → Universal Key
4. Generate or copy your key

## Post-Installation

### Create Admin User

```bash
cd /opt/wbuk-triage
./create-admin.sh
```

### Access MinIO Console

MinIO console is available at `http://your-server:9001`

Default credentials are in your `.env` file.

## Service Management

```bash
# Start all services
cd /opt/wbuk-triage
docker compose -f deploy/docker-compose.yml up -d

# Stop all services
docker compose -f deploy/docker-compose.yml down

# View logs
docker compose -f deploy/docker-compose.yml logs -f

# View specific service logs
docker compose -f deploy/docker-compose.yml logs -f backend

# Restart a service
docker compose -f deploy/docker-compose.yml restart backend

# Check status
docker ps
```

## SSL/HTTPS Setup

For production, you should set up SSL. Options:

### Option 1: Certbot (Let's Encrypt)

```bash
# Install certbot
apt install certbot python3-certbot-nginx

# Get certificate
certbot --nginx -d your-domain.com

# Auto-renewal is configured automatically
```

### Option 2: Cloudflare Proxy

1. Add your domain to Cloudflare
2. Set SSL mode to "Full"
3. Point DNS to your server

## Backup & Restore

### Backup

```bash
# Backup MongoDB
docker exec wbuk-mongodb mongodump --out /dump
docker cp wbuk-mongodb:/dump ./backup-$(date +%Y%m%d)

# Backup MinIO data
docker cp wbuk-minio:/data ./minio-backup-$(date +%Y%m%d)

# Backup environment
cp /opt/wbuk-triage/.env ./env-backup-$(date +%Y%m%d)
```

### Restore

```bash
# Restore MongoDB
docker cp ./backup-YYYYMMDD wbuk-mongodb:/dump
docker exec wbuk-mongodb mongorestore /dump

# Restore MinIO
docker cp ./minio-backup-YYYYMMDD wbuk-minio:/data
```

## Updating

```bash
cd /opt/wbuk-triage

# Pull latest code
git pull origin main

docker compose -f deploy/docker-compose.yml up -d --build
```

## Troubleshooting

### Check service health

```bash
docker ps
docker logs wbuk-backend
docker logs wbuk-frontend
```

### MongoDB connection issues

```bash
# Check MongoDB logs
docker logs wbuk-mongodb

# Test connection
docker exec -it wbuk-mongodb mongosh -u wbukadmin -p <password> --authenticationDatabase admin
```

### Backend not starting

```bash
# Check logs
docker logs wbuk-backend

# Common issues:
# - Missing EMERGENT_LLM_KEY
# - MongoDB not ready
# - Port conflicts
```

### Frontend not loading

```bash
# Check nginx logs
docker logs wbuk-frontend

# Rebuild frontend
docker compose -f deploy/docker-compose.yml build frontend
docker compose -f deploy/docker-compose.yml up -d frontend
```

## Security Recommendations

1. **Change default passwords** - Edit `.env` file
2. **Enable firewall** - Only allow ports 80, 443
3. **Use HTTPS** - Set up SSL certificate
4. **Regular backups** - Automated backup script
5. **Keep updated** - Regular security updates

```bash
# UFW firewall setup
ufw allow 22    # SSH
ufw allow 80    # HTTP
ufw allow 443   # HTTPS
ufw enable
```
